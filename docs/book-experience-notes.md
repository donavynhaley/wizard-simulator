# Book Experience Notes

Rune books are physical reference props, not inventory-style overlay readers.

The player should pick up a book from the world, open the book itself, and read page content on the book.

When held and opened, the book should move into a readable inspect pose in front of the camera.

The player should still be able to move while reading a held book.

Rune books should render rune template information onto the visible book pages.

When placed on the spellcrafting table, the book should stay open as an in-world reference.

During scribing, page turns should affect the table book itself so the player can use it while writing runes.

Avoid replacing this with a full-screen UI overlay unless the design explicitly changes.
